# logs — 실행 이력 (Evaluate 증거)

이 폴더에는 `docs/rules/workflow.md`의 "기능 로그" 규칙에 따라 **기능별 학습·검증 증거**가 쌓입니다.

- 파일 하나 = 기능 하나: `docs/logs/{기능}.md` (예: `point-charge.md`, `order.md`, `concurrency-test.md`)
- 작성법·템플릿은 [`docs/rules/workflow.md`](../rules/workflow.md)의 "기능 로그" 절 참고.

현재는 프로젝트 셋업 로그만 존재합니다. 이후 기능은 Plan 단계에서 파일을 만들고, Evaluate마다 실제 결과를 추가합니다.
